2. .github/workflows/build.yaml In GitHub, your automation files must live in a specific folder: .github/workflows/. This script tells GitHub to "build" or deploy your app (in this case, it simply acknowledges the code is pushed). name: Build and Deploy on: push: branches: [ main ] jobs: build: runs-on: ubuntu-latest steps: - name: Checkout code uses: actions/checkout@v4 - name: Run Build Script run: | echo "Building the Calculator App..." mkdir -p dist cp index.html dist/ echo "Build complete. Files are in the dist folder." How to use this: * Create a new Repository on GitHub. * Upload index.html to the main folder. * Create a folder named .github, then a folder inside it named workflows. * Upload build.yaml inside that workflows folder. Every time you "Push" a change to your code, GitHub will automatically run the "Build" job.
